## What are if statements?

If statements let your program make decisions. They check if something is true or false, then act accordingly.

Real-life example:
- IF it's raining, THEN take an umbrella
- IF battery < 20%, THEN charge phone
- IF password is correct, THEN allow access

## Basic if statement

The simplest decision:

```python
age = 18

if age >= 18:
    print("You can vote!")
    print("You're an adult")
```

How it works:
1. Check the condition (`age >= 18`)
2. If True, run the indented code
3. If False, skip it

<Note>
  The colon `:` and indentation are required! This is Python's syntax.
</Note>

## If-elif-else chains

For multiple conditions:

```python
score = 85

if score >= 90:
    print("A - Excellent!")
elif score >= 80:
    print("B - Good job!")
elif score >= 70:
    print("C - Keep it up!")
elif score >= 60:
    print("D - Need improvement")
else:
    print("F - Please study more")
```

Python checks each condition in order and runs the first True one.

## Comparison operators

These create True/False conditions:

```python
x = 10

# Equality
x == 10    # True (equal to)
x != 5     # True (not equal to)

# Greater/Less than
x > 5      # True
x < 20     # True  
x >= 10    # True (greater or equal)
x <= 10    # True (less or equal)

# Examples in if statements
if x > 5:
    print("x is greater than 5")

if x != 0:
    print("x is not zero")
```

## Multiple conditions

Use `and`, `or`, and `not`:

```python
age = 25
has_license = True

# Both must be True
if age >= 18 and has_license:
    print("You can drive!")

# At least one must be True
weekend = False
holiday = True
if weekend or holiday:
    print("No work today!")

# Reverse the condition
raining = False
if not raining:
    print("Let's go outside!")
```

### Complex conditions

```python
# Group with parentheses
temperature = 35
humidity = 80
rain = False

if (temperature > 30 or humidity > 70) and not rain:
    print("Hot and uncomfortable, but at least it's dry!")

# Multiple checks
password = "secret123"
if len(password) >= 8 and password.isalnum():
    print("Password is valid")
```

## Nested if statements

Put if statements inside other if statements:

```python
has_ticket = True
age = 15
has_guardian = True

if has_ticket:
    print("You have a ticket!")
    
    if age >= 18:
        print("Enjoy the movie!")
    else:
        if has_guardian:
            print("Enjoy the movie with your guardian!")
        else:
            print("Sorry, you need an adult with you")
else:
    print("Please buy a ticket first")
```

## Truthy and falsy values

Python considers these False:
- `False`
- `None`
- `0` (zero)
- `""` (empty string)
- `[]` (empty list)
- `{}` (empty dict)

Everything else is True:

```python
# Empty checks
name = ""
if name:
    print(f"Hello, {name}")
else:
    print("Please enter a name")

# List checks
items = []
if items:
    print(f"You have {len(items)} items")
else:
    print("Your cart is empty")

# Direct boolean check
is_logged_in = True
if is_logged_in:
    print("Welcome back!")
```

## Common patterns

### Guard clauses
Exit early to avoid deep nesting:

```python
def process_order(user, payment, items):
    # Check problems first
    if not user:
        return "No user"
    
    if not payment:
        return "No payment"
        
    if not items:
        return "No items"
    
    # Main logic here
    return "Order processed!"
```

### Ternary operator
One-line if-else:

```python
# Long way
if age >= 18:
    status = "adult"
else:
    status = "minor"

# Short way (ternary)
status = "adult" if age >= 18 else "minor"

# In print statements
print("Access granted" if authenticated else "Access denied")
```

### Switch-like behavior
Before Python 3.10's match-case:

```python
# Menu system
choice = input("Enter choice (a/b/c): ").lower()

if choice == 'a':
    print("Option A selected")
elif choice == 'b':
    print("Option B selected")
elif choice == 'c':
    print("Option C selected")
else:
    print("Invalid choice")
```

## Common mistakes

### Forgetting the colon
```python
# Wrong
if x > 5
    print("Big")

# Right
if x > 5:
    print("Big")
```

### Using = instead of ==
```python
# Wrong (assignment, not comparison)
if x = 5:
    print("Five")

# Right
if x == 5:
    print("Five")
```

### Wrong indentation
```python
# Wrong
if True:
print("Hello")  # IndentationError

# Right
if True:
    print("Hello")
```

### Empty if blocks
```python
# Wrong - can't have empty if
if condition:

# Right - use pass
if condition:
    pass  # TODO: add code later
```

## Best practices

1. **Keep conditions simple**
   ```python
   # Hard to read
   if x > 5 and y < 10 and z == 0 and not flag:
   
   # Better
   is_valid_range = x > 5 and y < 10
   is_ready = z == 0 and not flag
   if is_valid_range and is_ready:
   ```

2. **Use meaningful names**
   ```python
   # Bad
   if u.a >= 18:
   
   # Good
   if user.age >= 18:
   ```

3. **Avoid deep nesting**
   ```python
   # Too nested
   if a:
       if b:
           if c:
               if d:
                   do_something()
   
   # Better
   if not a:
       return
   if not b:
       return
   # etc.
   ```
